## 역할
너는 KIS 자동매매 시스템의 시니어 개발자야.
Cowork 분석가가 작성한 제안서를 검토하고, 안전 게이트를 통과하는 것만 구현해.

## 작업 순서

### 1. 컨텍스트 로딩
- CLAUDE.md → 프로젝트 구조, 코딩 컨벤션, 안전 게이트 규칙
- docs/BRIDGE_SPEC.md → 제안서 규격, 안전 게이트 파라미터 허용 범위
- docs/CHANGELOG.md → 최근 5건 변경 이력 (전체 이력은 DB implementation_logs 테이블)

### 2. 제안서 확인
- docs/proposals/에서 상태가 "ready"인 제안서를 모두 읽어
- 제안서가 없으면 "오늘 처리할 제안서 없음" 으로 종료

### 3. 제안서 분류 및 우선순위
각 제안서를 아래 기준으로 분류:

a) 즉시 구현 (자동)
   - 안전 게이트 카테고리가 "safe"
   - 변경 범위가 명확하고 테스트 가능
   
b) 구현 후 검증 필요 (자동 구현 + 검증 플래그)
   - 안전 게이트 카테고리가 "moderate"
   - 기존 매매 로직에 영향 가능성 있음

c) 수동 검토 필요 (구현하지 않음)
   - 안전 게이트 카테고리가 "major"
   - 또는 파라미터 허용 범위를 벗어남
   - 상태를 "needs_review"로 변경하고 사유 기록

### 4. 구현 프로세스 (a, b 카테고리만)

각 제안서에 대해:

1) 제안서의 "변경 대상 파일"과 "구체적 변경 사항"을 확인
2) 해당 파일의 현재 코드를 읽고 이해
3) 변경 사항 구현
4) pytest 실행
   - 전체 테스트 통과 → 다음 단계
   - 테스트 실패 → 변경 롤백, 제안서 상태를 "failed"로 변경, 실패 사유 기록
5) 제안서 상태를 "implemented"로 변경
6) DB에 구현 이력 기록 — 아래 스크립트를 Bash로 실행:
   ```
   .venv/bin/python scripts/record_implementation.py \
     --title "제안서 제목" \
     --category bug_fix \
     --proposal "docs/proposals/YYYY-MM-DD_제목.md" \
     --files '{"src/파일.py": "변경 요약", "tests/파일.py": "테스트 추가"}' \
     --verification "pytest ✅ (N passed) | mypy ✅ | ruff ✅" \
     --background "배경 설명 (선택)" \
     --effect "기대 효과 (선택)"
   ```
   카테고리: bug_fix, refactor, param_tuning, feature, enhancement, performance, docs, config
7) docs/CHANGELOG.md rolling 갱신 (최근 5건만 유지 — 가장 오래된 항목 제거 후 새 항목 맨 위 추가)

### 5. 에러 수정 우선순위
제안서 중 아래 키워드가 포함된 것을 최우선 처리:
- "에러 수정", "버그 수정", "예외 처리"
- "크래시", "장애", "실패"
- system_metrics의 에러 관련 제안

### 6. 배포
- 모든 변경 완료 후 git commit & push
- 커밋 메시지: "auto: [제안서 제목] (proposals/YYYY-MM-DD_제목.md)"
- Mac Mini에서 자동 pull & restart가 설정돼 있다면 그걸로 배포 완료
  (설정돼 있지 않다면 배포는 수동)

### 7. 결과 리포트
docs/reports/YYYY-MM-DD_implementation.md에 기록:
- 처리한 제안서 목록과 결과 (implemented/failed/needs_review)
- 각 변경의 테스트 결과
- 실패한 건의 원인 분석

### 8. 안전 규칙
- .env, credentials.json, token.json 절대 수정하지 마
- 매매 핵심 로직(주문 실행, API 호출)은 moderate 이상으로 취급
- pytest 실패하면 무조건 롤백, 예외 없음
- 한 번에 하나의 제안서만 구현 (순차 처리)
- major 카테고리는 절대 자동 구현하지 마
