## 현상 분석

04/18 이후 7일간 시그널이 DB에 0건 기록되었다. 코드 분석 결과 `src/engine.py:929-930`에서 HOLD 시그널은 DB 기록을 건너뛰고, `932-936`에서 저신뢰도 비실행 시그널도 건너뛴다.

이로 인해 **전략이 왜 HOLD를 반환하는지 진단할 수 있는 데이터가 전혀 남지 않는 상태**이다. 사이클당 스크리닝된 종목 수(10개) × 일일 사이클 수(1,500+) = 15,000+건의 전략 평가가 수행되지만, 그 결과가 모두 블랙홀로 사라지고 있다.

진단을 위해 사이클별 전략 평가 요약을 로깅하는 것을 제안한다. 모든 HOLD를 개별 기록하면 DB 부하가 과다하므로, **사이클 단위 집계 로그**를 추가한다.

## 제안 내용

매매 사이클 종료 시점에 해당 사이클에서 평가한 전략 결과의 요약 통계를 INFO 레벨로 로깅한다.

## 변경 스펙

### 파일별 변경사항

- `src/engine.py`:
  - `_run_trading_cycle()` 메서드 내, 종목별 전략 평가 루프 직후에 사이클 요약 로그 추가:
    ```python
    logger.info(
        "사이클 #%d 전략 요약: 평가 %d종목, BUY %d / SELL %d / HOLD %d, "
        "max_confidence=%.3f, 스크리닝 %d종목",
        cycle_number,
        total_evaluated,
        buy_count,
        sell_count,
        hold_count,
        max_confidence,
        screened_count,
    )
    ```
  - 구현 방식: 루프 내에서 `buy_count`, `sell_count`, `hold_count`, `max_confidence` 카운터를 유지하고 루프 종료 후 한 줄 로그 출력
  - **기존 로직 변경 없음**: 카운터 변수 추가와 로그 한 줄 추가만 수행

### 추가 테스트 (필요 시)

- 기존 `tests/test_engine_db_integration.py`에 사이클 요약 로그 출력 확인 테스트 추가 (선택)
- 핵심은 기존 테스트가 깨지지 않는 것이므로, 로그 추가만으로는 별도 테스트 불필요할 수 있음

## 기대 효과
