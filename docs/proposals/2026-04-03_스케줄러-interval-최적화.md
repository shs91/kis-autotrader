## 현상 분석

### 데이터
- `logs/autotrader.err.log` 4/1일 기준: "maximum number of running instances reached" 에러가 12:36~15:16 동안 약 5분 간격으로 반복 발생 (총 29회 이상)
- 장중 매매 사이클 interval: 10초
- 관심종목 수: 7개 (005930, 000660, 035420, 012450, 034020, 207940, 105560)

### 원인 분석
`_calculate_trading_interval()` 함수의 계산:
```
calls_per_cycle = 7종목 × 2건(일봉+현재가) + 1건(잔고) = 15건
interval = ceil((15 / 3) × 1.2 × 10) / 10 = 6.0초
max(6.0, 10.0) = 10.0초 (하한값 적용)
```

실제 소요 시간은 네트워크 지연 + 스크리닝 오버헤드로 10초를 초과.
`max_instances=1`이라 스킵되며, 장중 내내 반복 발생.

## 제안 내용

interval 하한값을 10초 → 25초로 상향하여 일반/스크리닝 사이클 모두 커버.
이동평균 교차 전략 특성상 25초 간격이어도 시그널 포착에 영향 미미.

## 변경 스펙
