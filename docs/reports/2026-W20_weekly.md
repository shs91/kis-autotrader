## 중기 아키텍처 논의

### 1. MACD 영구 비활성화 — 페이지네이션 제안서 미달성 (재발견)

**W19에서 식별 → W20에서 동일 문제 재확인됨.** 페이지네이션 제안서(`2026-05-09_daily-quote-pagination-60days.md`, 5/11 구현)가 적용되었음에도 5/15 SIGNAL_SKIP 표본 전건이 `series_len: 30`이며 MACD `guard_triggered: true`. 코드(`src/api/quote.py:144`)는 60건 lookback으로 페이지네이션 루프를 도는 구조이나, **KIS `inquire-daily-price` 엔드포인트가 `FID_INPUT_DATE_1/2` 파라미터를 무시하고 항상 최근 30거래일만 반환하는 한계**로 추정됨. 두 번째 페이지 결과가 첫 페이지와 동일하여 `dt in seen_dates` 체크로 모두 스킵되고, `len(output_list) < 30` 조건으로 루프 종료.

→ **해결책**: `inquire-daily-itemchartprice` 엔드포인트(최대 100건, 기간 지정 가능)로 전환. 신규 제안서 A 작성.

### 2. 단일 종목 데이트레이딩 집중 — LG디스플레이 5/14 3회전

5/14 매수 3건 전부 LG디스플레이(034220) 동일 종목. 매매 6건(매수 3 + 매도 3)으로 주간 매매의 24%를 한 종목이 차지. 종목 선택 다양성 결여 가능성 — 스크리닝이 정상 작동하나 엔진의 종목 선정 로직이 보유 종목 재진입에 편향됐을 수 있음. 단, 결과적으로는 손익비 1.3으로 양호 — 단기적으로는 문제 아님.

→ **2주 추가 관찰**. 동일 종목 ≥3회 진입이 W21에도 반복되면 종목 일별 최대 진입 횟수 제한(예: `MAX_DAILY_TRADES_PER_STOCK=2`) 도입 논의.

### 3. `action_taken=true` 의미 모호성 — 5/15 321건 시그널이 매수 미발생

5/15 시그널 1,092건 중 acted=true 336건, 그러나 실제 매수 0건. 종목 021880이 364건 시그널 중 336건 acted=true(92%), 모두 "앙상블 가중투표: BUY 1.00 vs 0.00", confidence 0.25 — 매우 강력한 BUY 시그널인데 실제 주문 미발생. 코드 분석 결과 `_record_signal_to_db(...action_taken=True)`는 위험 게이트(`risk.validate_order`) 통과 후 수량 계산 직전에 호출 — 잔고 부족 또는 다른 차후 게이트에서 실패한 시그널은 acted=true로 기록되되 실제 주문이 발생하지 않음. **분석 자동화의 신뢰성에 영향**: act_rate 수치가 "실제 매매 발생 비율"이 아니라 "리스크 게이트 통과 비율"임.

→ **해결책**: `action_taken` 의미 명확화. 옵션 (a) `signals.actually_traded` 컬럼 추가(주문 체결 시 True 갱신), (b) 현재 `action_taken`을 "주문 시도(`order_attempted`)"로 리네이밍하고 분리. 단기적으로는 분석 SQL에서 `signals JOIN trades` 기준으로 진정한 act_rate 별도 산출 권장. 본 항목은 W21 일일 리포트에 후속.

### 4. 스크리닝 전환율 0%의 본질 — `converted_to_trade` 플래그 의미 차이
