# 발표 자료 — AI를 운영하는 개발자

Slidev 기반 발표 자료입니다. 약 33장 슬라이드, 45–60분 분량.

## 빠르게 띄우기

```bash
# 1. Slidev 설치 (전역)
npm install -g @slidev/cli

# 2. 슬라이드 폴더에서 실행
slidev slides.md

# 브라우저에서 자동으로 열림 (http://localhost:3030)
```

또는 npx로 즉시 실행:

```bash
npx @slidev/cli slides.md
```

## 발표용 단축키

| 키 | 동작 |
|---|---|
| `→` / `Space` | 다음 슬라이드 |
| `←` | 이전 슬라이드 |
| `f` | 풀스크린 |
| `o` | 슬라이드 오버뷰 |
| `d` | 다크/라이트 토글 |
| `g` | 특정 슬라이드로 점프 |

## 발표자 모드

```bash
slidev slides.md --presenter
```

→ 한 화면엔 슬라이드, 다른 화면엔 노트와 다음 슬라이드.

## PDF 내보내기

```bash
slidev export slides.md
# → slides-export.pdf 생성
```

## 슬라이드 구성

- **Part 1** (3장): 오프닝 — 새벽 3시 시스템
- **Part 2** (8장): 3단계 진화 (Lv1 → Lv2 → Lv3)
- **Part 3** (8장): 실전 케이스 — 자율 운영 파이프라인
- **Part 4** (7장): 전이 가능한 패턴 5가지
- **Part 5** (5장): 개발자의 새로운 역할
- **Closing** (2장): 마무리 + Q&A

## 커스터마이징 포인트

발표 직전에 손볼 부분들:

1. **첫 슬라이드 타이틀 하단**: 회사명/날짜 — 사내 발표 톤에 맞게 조정
2. **데모 슬라이드 (Part 2)**: 실제 `proposal_*.md` 스크린샷으로 교체하면 임팩트 ↑
3. **아키텍처 다이어그램**: ASCII로 그려뒀지만, 시간 되면 Excalidraw나 Mermaid로 교체 권장
4. **Part 3 패턴 섹션**: 사내 사례를 한두 줄씩 추가하면 청중 공감대 ↑

## 한국어 폰트

`fonts:` 설정에 Pretendard(본문)와 Noto Serif KR(제목 강조)을 지정해뒀습니다. 로컬에 설치되어 있지 않으면 자동으로 Google Fonts에서 받아옵니다.

## 다크 테마 톤

- 배경: `#0a0a0f` (거의 검정, 살짝 보라 기운)
- 강조 1: `orange-400` (현재 상태/문제 제기)
- 강조 2: `cyan-400` (도구/구현)
- 강조 3: `emerald-400` (지향점/긍정)
- 강조 4: `amber/red-400` (안전성 게이트)

이 색 매핑이 발표 전체에서 일관되게 쓰입니다.
